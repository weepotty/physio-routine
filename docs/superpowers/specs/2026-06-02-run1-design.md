# Run 1 — Design Spec

**Date:** 2026-06-02 (updated with warm-up 2026-06-02)

## Overview

Add a "Run 1" option to the home screen of the physio app. The session begins with a 4-exercise warm-up (30 seconds each), then moves into 8 rounds of 1-minute half-speed run + 2-minute walk (24 minutes). Total session ~26 minutes. Audio cues announce each phase change.

## Home Screen Changes

- Hero subtitle changes from "Two-day programme" to "Physio & running programme".
- Third card added: green pill labelled "Run", title "Run 1", subtitle "Half-speed intervals · 24 min".
- Tapping the card calls `startRun()` which shows the run screen directly (no list screen).

## Run Screen (`run-screen`)

New fourth screen alongside `day-screen`, `list-screen`, `exercise-screen`. Toggled via `showScreen()`.

### Layout — Warm-up phase

```
[ ← back ]              Warm-up 2 of 4

        HIGH KNEES
           0:22
     Up next: Hip openers

         [ Pause ]
```

### Layout — Interval phase

```
[ ← back ]                    Round 3 of 8

           RUN
           0:42
      Walk next · 2:00

         [ Pause ]
```

- Back arrow → `goToDayPicker()` (clears run timer)
- Phase label: exercise name (amber) during warm-up; "RUN" (green) or "WALK" (grey-blue) during intervals
- Countdown: `M:SS` format
- Next-phase hint: next exercise name, or "Run next · 1:00" after last warm-up, or upcoming interval phase
- After round 8 walk completes: phase label becomes "Done!", countdown hidden, button becomes "Start over"
- Single action button: Start / Pause / Resume

### Visual style

Follows existing card pattern. Phase colours:
- WARM-UP: `#d97706` (amber)
- RUN: `#16a34a` (green)
- WALK: `#1a7fa6` (blue-grey)

## Session Logic

### Warm-up (mode: `"warmup"`)

4 exercises, 30 seconds each, auto-advancing:
1. Stretches
2. High knees
3. Hip openers
4. Broad jumps

### Intervals (mode: `"intervals"`)

8 rounds. Each round = 60s run + 120s walk. Round 8 walk plays out fully before "Done!".

### State machine

`idle → running (warmup) → ... → running (intervals) → paused → running → ... → done`

Phase transitions happen automatically when countdown reaches zero. Pause/resume works in both modes.

### State variables

| Variable | Type | Purpose |
|---|---|---|
| `runState` | `"idle" \| "running" \| "paused" \| "done"` | Controls button label and behaviour |
| `runMode` | `"warmup" \| "intervals"` | Which stage of the session |
| `warmupIndex` | number (0–3) | Current warm-up exercise |
| `runPhase` | `"run" \| "walk"` | Current interval phase (intervals mode only) |
| `runRound` | number (1–8) | Current interval round |
| `runRemainingTime` | number | Seconds left in current phase |
| `runIntervalId` | number \| null | `setInterval` handle |

## Audio

### Primary: Web Speech API

`speakPhase(phase)` uses `window.speechSynthesis`. Known phases:
- `"run"` → "Run!"
- `"walk"` → "Walk!"
- `"done"` → "Well done!"
- Any other string (exercise names) → appended with "!" e.g. "High knees!"

`speechSynthesis.cancel()` is called before each `speak()` to prevent stacked announcements.

### Fallback: Web Audio tones

- `"walk"` → `playWalk()` (descending 880→660 Hz)
- `"done"` → `playAllSetsComplete()`
- All other phases → `playGo()` (ascending 880→1175 Hz)

## Navigation

- Home → Run 1 card → `startRun()` → `run-screen` (warm-up, idle)
- Back button on run screen → `goToDayPicker()` → `day-screen`
- "Start over" → resets to warm-up idle, stays on run screen
