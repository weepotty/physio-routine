# Run 1 — Design Spec

**Date:** 2026-06-02

## Overview

Add a "Run 1" option to the home screen of the physio app. It is a continuous interval running session: 8 rounds of 1-minute half-speed run followed by a 2-minute walk (24 minutes total). Audio cues announce each phase change.

## Home Screen Changes

- Hero subtitle changes from "Two-day programme" to "Physio & running programme".
- Third card added: green pill labelled "Run", title "Run 1", subtitle "Half-speed intervals · 24 min".
- Tapping the card calls `startRun()` which shows the run screen directly (no list screen).

## Run Screen (`run-screen`)

New fourth screen alongside `day-screen`, `list-screen`, `exercise-screen`. Toggled via `showScreen()`.

### Layout

```
[ ← back ]                    Round 3 of 8

           RUN
           0:42
      Walk next · 2:00

         [ Pause ]
```

- Back arrow → `goToDayPicker()` (clears run timer)
- Phase label: "RUN" (green) or "WALK" (grey-blue), large and bold
- Countdown: `M:SS` format
- Next-phase hint: small muted text showing the upcoming phase and its duration
- After round 8 walk completes: phase label becomes "Done!", countdown hidden, button becomes "Start over"
- Single action button: Start / Pause / Resume

### Visual style

Follows existing card pattern (`background: white`, `border-radius: 20px`, `box-shadow`). Phase colours:
- RUN: `#16a34a` (green)
- WALK: `#1a7fa6` (blue-grey, matches existing day-pill-2)

## Session Logic

- 8 rounds. Each round = 60s run + 120s walk.
- Round 8 walk phase plays out fully before showing "Done!".
- State machine: `idle → running → paused → running → ... → done`
- Phase transitions happen automatically when countdown reaches zero.

### State variables

| Variable | Type | Purpose |
|---|---|---|
| `runState` | `"idle" \| "running" \| "paused" \| "done"` | Controls button label and behaviour |
| `runPhase` | `"run" \| "walk"` | Current phase |
| `runRound` | number (1–8) | Current round |
| `runRemainingTime` | number | Seconds left in current phase |
| `runIntervalId` | number \| null | `setInterval` handle |

## Audio

### Primary: Web Speech API

- Detected at phase transition: `window.speechSynthesis` availability check.
- `speechSynthesis.speak(new SpeechSynthesisUtterance("Run!"))` at start of each run phase.
- `speechSynthesis.speak(new SpeechSynthesisUtterance("Walk!"))` at start of each walk phase.
- `speechSynthesis.speak(new SpeechSynthesisUtterance("Well done!"))` on session complete.

### Fallback: Web Audio tones (if `speechSynthesis` unavailable)

- Run cue: `playGo()` (existing — two ascending notes).
- Walk cue: new `playWalk()` function — two descending notes (880 Hz → 660 Hz).
- Done cue: existing `playAllSetsComplete()`.

Both voice and tone cues play at phase start only (no countdown warning tones).

## Navigation

- Home → Run 1 card → `startRun()` → `run-screen`
- Back button on run screen → `goToDayPicker()` → `day-screen`
- "Start over" button on done state → resets all run state, stays on run screen in idle
